Posted on 2019-06-20

九把刀在书里说过：“有些梦想，纵使永远也没办法实现，纵使光是连说出来都很奢侈。但如果没有说出来温暖自己一下，就无法获得前进的动力。”

人为什么要背负感情？是因为只有在人们面对这些痛楚之后，才能变得强大，才能在面对那些无能为力的自然规律的时候，更好地安慰他人。人为什么要背负梦想？是因为梦想这东西，即使你脆弱地随时会倒下，也没有人能夺走他。即使你真的是一条咸鱼，也没人能夺走你做梦的自由。

所有的辉煌和伟大，一定伴随着挫折和跌倒，所有的牛逼背后都是一座座苦逼的高墙。谁没有一个不安稳的青春？没有一件事情可以一下子把你打垮，也不会有一件事情可以让你一步登天，慢慢走，慢慢看，生命是一个慢慢累积的过程。

软件项目管理复习

项目

相关概念

项目

项目是一个特殊的、将被完成的有限任务，它是在一定时间 内，满足一系列特定目标的多项相关工作的总称

项目的生命周期

启动、计划、实施、结束

基线

指一个（或一组）配置项在项目生命周期的不同时间点上通过正式评审而进入受控的一种状态

检查点

在规定的时间间隔内对项目进行检查，比较实际与计划之间的差异，并根据差异进行调整

软件设计

把 软件需求 变换成为 软件表示 的过程。是开发阶段的重要步骤，也是后续软件开发和软件维护的基础

质量

一组固有特性满足要求的程度

软件质量策划是质量管理的一部分，致力于设定 质量目标 并规定必要的 运行过程 和 相关资源 以实现质量目标

质量控制
  • 监测
    • 监测是收集、记录和汇报有关项目质量信息
  • 控制
    • 质量监测提供的数据进行控制，确保项目质量与计划保持一致
    • 软件产品或服务的质量控制
    • 软件管理过程的质量控制

投资回收期

静态投资回收期

用项目各年的净收入将全部投资回收所需要的年限

动态投资回收期

在考虑资金时间价值条件下，按设定的利率回收全部投资所需要的时间

可行性研究

报告

  • 分析论证项目投资建设的必要性
  • 分析项目投资建设的可行性
  • 分析项目投资建设的合理性

内容

  • 技术上的可行性
  • 经济上的可行性
  • 社会上的可行性

需求分析

需求类别

  • 业务需求
  • 用户需求
  • 功能需求

实现步骤

  1. 获得当前系统的物理模型
  2. 抽象出当前系统的逻辑模型
  3. 导出目标系统的逻辑模型
  4. 最终目标系统的物理模型是由逻辑模型经实例化，即具体到某个业务领域而得到的

任务

借助于当前系统的逻辑模型导出目标系统的逻辑模型，解决目标系统“做什么”的问题

内容

  • 功能需求

  • 性能需求

    • 精度

    • 时间特性要求

    • 灵活性

  • 输入输出要求

  • 数据管理能力要求

  • 环境需求

  • 可靠性需求

  • 安全保密要求

  • 用户界面需求

  • 资源使用需求

  • 软件成本消耗与开发进度需求

重要性

  • 需求分析是获得用户需求的有效途径
  • 需求分析是决定项目成功的关键因素
  • 需求分析是系统分析和软件设计的桥梁
  • 需求分析是控制软件质量的重要阶段

变更原因

  • 问题域的复杂性越来越高
  • 交流障碍
  • 完整性问题
  • 变更的需求引起更多的变更

软件生命周期

含义

指从设计软件产品开始到软件产品不能再使用为止的时间周期

每一阶段工作以 上一阶段 工作的结果为依据，并为 下一阶段 工作提供基础

  • 设计阶段

  • 测试阶段
  • 安装和验收阶段
  • 运行和维护阶段

  • 外部接口

软件过程管理

  • 客户—供应商过程

测试

白盒

也称 结构测试 或 逻辑驱动测试 ，是一种按程序内部的逻辑结构和编码结构设计测试数据的测试方法。主要用于软件验证

黑盒

也称 功能测试 或 数据驱动测试 ，是已知产品所应具有的功能，通过测试来检测每个功能是否都正确时使用。在测试时，把程序看成是一个不能打开的黑盒子，检查程序功能是否按照需求规格说明书的规定正常使用

软件测试过程

  • 单元测试
    • 模块接口
    • 局部数据结构
    • 边界条件
    • 独立路径
    • 出错处理
  • 集成测试
  • 确认测试
    • 功能测试
    • 性能测试
    • 强度测试（压力测试）
    • 安全性测试
  • 系统测试

软件设计规格说明书

  • 详细设计
  • 接口设计
  • 数据库设计
