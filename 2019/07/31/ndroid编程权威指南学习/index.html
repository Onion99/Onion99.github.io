Posted on 2019-07-31 | In Android

Fragment

采用fragment来管理应用UI,可避开Android系统activity使用规则的限制,顾名思义这是歪果仁

fragment是一种控制器对象,activity可委派他执行任务

UI fragment 是管理用户界面的fragment,fragment 视图包含了用户可以交互的可视化UI元素

根据应用和用户需求,可联合使用fragment及Activity来组装或重组用户界面

生命周期

类似于Activity,关键在于,fragment的生命周期方法由Activity托管,系统不关心activity用来管理视图的fragment. fragment的使用是activity内部的事情

托管的两种方式

在Activity布局中使用fragment

简单但不灵活,在Activity的生命周期过程中,无法更换fragment视图

在Activity代码中使用fragment

动态控制

定义容器视图

定义通用性的视图,以托管更多的fragment

创建UI Fragment

注意

  • Fragment.onCreate(Bundle)是公共方法，Activity.onCreate(Bundle)是受保护法.Fragment.onCreate(Bundle)方法及其他Fragment生命周期方法必须是公共方法.
    因为托管fragment的activity要调用他们
  • 类似于Activity,Fragmen同样具有保存及获取状态的bundle.如同使用Activity.onSaveInstanceState(Bundle)方法那样,也可以根据需要覆盖Fragment.onSaveInstanceState(Bundle)方法
  • Fragment视图的创建和配置在onCreateView()中完成,其方法接收三个参数
    1. 视图直接通过调用LayoutInflater.inflate() 方法传入布局的资源ID生成
    2. 视图的父视图,通常需要父视图来正确配置组件
    3. 告诉布局生成器是否将生成的视图添加给父视图
      • if (fragment == null)流程:向FragmentManager请求并获取fragment,如果要获取的fragment在队列中,FragmentManager就直接返回它,之前有可能存在?
      • 设备旋转或回收内存时,Android洗头会销毁CrimeActivity,而后重逢时,会调用CrimeActivity.onCreate(Bundle)方法,因为activity销毁时,他的FragmentManager会将fragment队列保存下来

FragmentManager

fragment 事务

fragment事务被用来添加,移除,附加,分离,或替换fragment队列中的fragment.这是使用fragment动态组装和重新组装用户界面的关键

fm.beginTransaction().add(R.id.fragment_container, fragment).commit();
* beginTransaction()方法创建并返回 FragmentTransaction实例, FragmentTransaction类支持流接口的链式调用,因此以上代码可以理解为:创建一个新的 fragment事务,执行一个fragment添加操作,然后提交该事务* add()是整个事务的核心,其接受两个参数

  • 容器资源Id
    • 告诉FragmentManager,fragment的视图应该在Activity视图的什么位置
    • 唯一标识FragmentManager队列中的fragment
  • 要设置的Fragment

单例与数据集中存储

应用能在内存里活多久,单例就能活多久,值得注意的是,Android从内存里清楚应用时,单例对象也会消失. 虽然单例CrimeLab单例不是数据持久保存的好方案,但它确实能保证仅拥有一份CirmeLab数据,并且能让控制器曾类间的数据传递更容易

ConstraintLayout

位置依赖于约束

布局属性

sp,dp,以及屏幕像素密度

  • px,即像素,一个像素单位对应一个屏幕像素单位,不会根据屏幕密度自动缩放
  • dp,即密度无关像素,自动缩放
  • sp,即缩放无光像素,受用户字体偏好设置的影响,通常用于设置屏幕字体

    边距(margin)与内边距(padding)

    • 编剧决定组件的距离
    • 内边距决定组件距离所包含内容的大小

样式(style),主题(theme),主题属性

  • 样式,是对组件行为和外观的属性定义
  • 主题,即各种样式的集合,从结构上主题本身也是一种样式资源,只不过它的属性指向了其他样式资源

Android应用的设计原则

注意看边距属性，==Android Studio默认使用的值是16dp或8dp。设定这两种值遵循了Android 的material design原则==,Android设计规范

Fragment argument

fragment有两种获取Intent中数据的方式,一种通过getIntent.get***,简单粗暴,但这会破坏fragment的封装,另一种就是Fragment argument

每一个fragment实例都可附带一个Bundle对象,该bundle包含键-值对,可以像activity中的intent中的那样使用它们,一个键值对即一个argument
