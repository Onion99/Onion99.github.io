访问级别

  • 在同一个包中，所有的类都能够访问某一个类中所有非private权限的方法或者属性。
  • 一个包中的类不能通过继承访问另一个包中default权限的方法或者属性。换言之,子类在同一个包中可以访问default权限的方法或属性，子类不在同一个包中则不可以
  • 在不同包中子类是不能通过父类的实例访问protected权限的方法或者变量的
  • 如果不允许外部直接通过new创建对象,构造方法必须是private
  • 工具类不允许有public或default构造方法
  • 类非static成员变量并且与子类共享,必须是protected
  • 类非static成员变量并且仅在本类使用,必须是private
  • 类static成员变量如果仅在本类使用,必须是private
  • 若是static成员变量,必须考虑是否为final
  • 类成员方法只供类内部调用,必须是private
  • 类成员方法只对继承类公开,那么限制为protected

组成

  • 类型本身具有的属性,通过类变量
  • 类型本身可以进行的操作,通过类方法体现
  • 类型实例具有的属性.通过实例变量
  • 类型实例可以进行的操作,通过实例方法体现

  • 每new一次,就会产生一个对象,就会有一份独立的实例变量

  • 对象的释放是被Java垃圾回收机制管理的

构造方法

  • 名称是固定的,与类名相同
  • 没有返回值,也不能有返回值.构造方法隐含的返回值就是实例本身
  • 构造方法不能被继承,不能被覆写,不能被直接调用
  • 默认构造方法
    • java编译器会自动生成一个默认构造方法,但一旦定义构造方法,Java就不会自动生成默认的
  • 私有构造方法,应用场景
    • 不能创建类的实例,类只能被静态访问,如Math和Arrays类
    • 能创建类的实例,但只能被类的静态方法调用 . 如在单例中,对象是用过静态方法获取的,而静态方法调用私有构造方法
    • 只是用来被其他构造方法调用,用于减少重复代码

实例方法

静态方法

通常静态方法用于定义工具类的方法等,静态方法如果使用了可修改的对象,那么在并发时会存在线程安全问题.所以工具类的静态方法与单例通常是相伴而生

  • 静态方法中不能使用实例成员和实例方法
  • 静态方法不能使用super和this关键字,这两个关键字指代的都是需要被创建出来的对象

静态代码块

是先于构造方法执行的特殊代码块

  • 在类加载的时候调用,并且执行一次
  • 静态代码块不能存在于任何方法体内,包括类静态方法和属性变量

类的组合

分解现实的问题中涉及的概念及概念之间的关系,将概念表示为多个类,通过类之间的组合表达更为复杂的概念以及概念间的关系,是计算机程序的一种基本思维方式

图形类

设计线时,我们考虑的层次是点,而不考虑点的内部细节.每个类封装其内部细节,对外提供高层次的功能,使其他类在更高层次上考虑问题和解决问题,是程序设计的一种基本思维方式

用类描述电商概念

想想现实问题有哪些概念,这些概念有啥属性,哪些行为,概念之间有什么关系,然后定义类,定义属性,定义方法,定义类之间的关系. 概念的属性和行为可能是非常多的,但定义的类只需要包括那些与现实问题相关的就行了

描述人之间的血缘关系

一个类定义中还可以引用自己,在像文件与目录,两个类之间还可以相互调用

类的继承

使用继承一方面可以复用代码,公共的属性和行为可以放到父类中,而子类只需要关注子类特有的就可以了;另一方面,不同子类的对象可以更为方便地被统一处理

  • 每个父类有且只有一个父类,没有声明父类的,其父类为Object,子类继承父类非private的属性和方法,可以增加自己的属性和方法,以及重写父类的方法实现
  • new过程中,父类先进行初始化,可通过super调用父类的构造方法,没有super的情况下,调用父类的默认构造方法
  • 子类变量和父类重名的情况下,可通过super强制访问父类的变量和方法

构造方法细节

  • 当父类只有一个带参数的构造的方法且没有默认构造方法时,他的任何子类都必须在构造方法中通过super调用Base的带参数构造方法,否则报错
  • 父类构造方法调用可被重写的方法,则可能会出现意外的结果
    • 在父类的构造方法中调用可被子类重写的构造方法,是一种不好的实践,容易起混淆,应该只调用private的方法

重名与静态绑定

  • 重名的类型
    • private
      • 变量和方法只能在类内访问,访问的也永远是当前类
    • public
      • 类内,访问的是当前类,子类可通过super访问父类
      • 类外,则看静态类型,父则父,子则子
  • 静态绑定
    • 实例变量,静态变量,静态方法,private方法,都是静态绑定,即访问绑定到变量的静态类型
    • 静态绑定在程序编译阶段即可决定,而动态绑定则要等到程序运行时

重载和重写

重写指方法名同而参数不同(个数,顺序,类型),重写是指子类重写与父类相同参数签名的方法

当有多个重名函数的时候,在决定要调用哪个函数的过程中.首先是按照参数类型进行匹配的,换句话说,寻找在所有重载版本中最匹配的,然后才看变量的动态类型,进行动态绑定

父子类型转换

即向下转型,语法上可以进行强制类型转换,但不一定能转换成功

使用instanceof进行安全类型转换,instanceof前面是变量,后面是类,返回是boolean值,表示变量引用的对象是不是该类或其子类的对象

继承访问权限protected

protected,表示虽然不能被外部任意访问,但可被子类访问,另外protected还表示可被同一个包中的其他类访问,不管该类是不是该类的子类


基类定义对外行为的方法action,并定义了可以被子类重写的两个步骤step1()和step2()以及被子类查看的变量currentStep,子类通过重写protected方法step1()和step2()来修改对外行为.

这种思路和设计是一种设计模式,称之为模板方法,action方法就是个模板方法,它定义了实现的模板,而具体实现则由子类来提供.模板方法在很多框架中有广泛的应用,这是protected的一种常见场景

可见性重写

即重写时,子类方法不能降低父类方法的可见性,父类如果是public,则子类必须是public,父类如果是protected,子类可以protected,也可以是public,即可见性由低到高是: private < public< protect

规定原由: 继承反映的是” is - a “的关系,即子类对象也属于父类,子类必须支持父类所有的对外的行为,将可见性降低就会减少子类对外的行为,从而破坏”is - a “ 关系

防止继承final

继承破环封装(主要是从封装的原则上理解,看个人使用情况吧)

子类和父类之间存在实现细节的依赖,子类重写父类的方法可能导致错误结果(因而重写了,最好不再super去调用父类方法了)

  • 对于子类而言,通过继承实现是没有安全保证,因为父类修改内部实现细节,他的功能就可能会被破环
  • 而对于基类而言,让子类继承和重写方法,就可能丧失随意修改内部实现的自由
  • 继承没有反映is-a关系

避免破环封装

使用final避免继承
  • 给方法加final修饰符,父类就保留了随意修改这个方法内部实现的自由,使用这个方法的子类也可以确保其行为符合父类声明
  • 给类加修斯符,父类就保留了随意修改这个类实现的自由.使用者可以放心的使用,而不用担心一个父类引用的变量,实际指向的却是一个完全不符合预期行为的子类对象
优先使用组合而非继承

使用组合可以抵挡父类变化对子类的影响,从而保护子类,应该优先使用组合

组合的问题:子类对象不能当作基类对象来统一处理,解决方法是使用接口

正确使用继承
  • 基类他写,我写子类
    • 重写方法不要改变预期的行为
    • 阅读文档说明,理解可重写方法的实现机制,尤其是方法之间的依赖关系
    • 在基类修改的情况下,阅读其修改说明,相应修改子类
  • 基类我写,他写子类
    • 使用继承反映真正的is-a关系,只将真正公共的部分放到基类
    • 对不希望被重写的公开方法添加final修饰符

类的接口

接口声明一组能力,却没有实现,只是一个约定,接口涉及交互双方,一方需要实现这个接口,另一方使用接口(类似一个脚本精灵,一方使用脚本,一方编写脚本)

通常使用接口对行为进行抽象(转自LoulanPlan)

定义接口(interface)

  • interface,声明接口,修饰符一般都是public
  • java8之前,接口内部不能实现方法.接口方法不需要修饰符,加不加都是public abstract

实现接口(implements)

  • implements,表示实现接口,前面类名,后面接口名
  • 一个类可以实现多个接口,表明类的对象具备多种能力,接口间以逗号分隔

使用接口

  • 针对接口而非具体类型进行编程,是计算机程序的一种重要思维
  • 接口提升了代码复用,更重要的是降低了耦合提高了灵活性
  • 解决问题的核心就是大题分解为小题,分解的核心就是要降低耦合,提高灵活性

抽象类

抽象类与接口有些相似,但根本上是不同的,接口中不能定义实例变量,而抽象类可以,一个类可以实现多个接口,但只能继承一个类
类是客观事物的抽象，而抽象类又是类的进一步抽象,即抽象类是用来捕捉子类的通用特性的，包括属性及行为

如宝马,本田,奔驰,这些汽车都有通用的属性和行为,如品牌,发动机等属性,前进,后退等行为,在这些基础上可以抽象出”汽车类”,来包含通用的特性,让不同的汽车类继承这个抽象类,便有了汽车的通用特性,然后在此基础上定义自己特殊的属性和方法

接口与抽象类的区别

抽象类和接口经常相互配合,接口定义能力,而抽象类提供默认实现,方便子类实现接口

  • 抽象类是对类本质的抽象，表达的是 is a 的关系，比如：Audi is a Car。抽象类包含并实现子类的通用特性，将子类存在差异化的特性进行抽象，交由子类去实现
  • 接口是对行为的抽象，表达的是 like a 的关系。比如：Audi like a Aircraft（像飞行器一样可以飞），但其本质上 is a Car。接口的核心是定义行为，即实现类可以做什么，至于实现类主体是谁、是如何实现的，接口并不关心

内部类

内部类可以方便地访问外部类的私有变量,可以声明为private从而实现对外完全隐藏,相关代码写在一起,写法也更为简洁,这些都是内部类的好处
内部类只是相对于编译概念而言,于Java虚拟机来说,每个内部类最后都会被编译为一个独立的类

静态内部类

适用于与外部类关系密切,却不依赖于外部类实例的场景

成员内部类

适用于需要外部类的实例变量或方法,则考虑定义为成员内部类

匿名内部类

匿名内部类没有单独的类定义,他在创建对象的同时定义类,适用于对象只会创建一次,且不需要构造方法来接收参数的情景,还经常用于事件处理过程中(回调)

将程序分为保持不变的主体架构,和针对具体情况的可变逻辑,通过回调的方式进行协作,是计算机程序的一种常用实践.匿名内部类是实现回调接口的一种简单方式

寻找要执行的实例方法的时候,是从对象的实际类型信息开始查找的,找不到的时候,再查找父类信息

以c.action()为例

  • 查看c的对象类型,找到Child类型,在此类查找方法,发现没有,到父类寻找
  • 在父类Base找到,执行方法action()
  • action先输出start,然后发现需要调用step(),因为要判断是否重写了,就从子类Child寻找
  • 在子类Child找到step方法,执行,执行完返回action方法
  • 继续执行action方法,输出end

优化调用效率的虚方法表即在类加载的时候为每个类创建一个表,记录该类的对象的所有动态绑定的方法(包括父类)及其地址,但一个方法只有一条记录,子类重写父类方法后只会保留子类的

分类

try代码块

  • try 代码块
    • 监视代码执行过程,一旦发现异常则直接跳转至catch,如果没有catch,则跳转至finally
  • catch 代码块
    • 可选执行的代码块,如果没有任何异常则不会执行,如果发现异常则进行处理或向上抛出
  • finally 代码块
    • 必选执行的代码块,不管是否有异常产生,即使发生OOM也会执行,通常用于善后工作,如果fianlly代码块没有执行那么有三种可能
      • 没有进入try代码块
      • 进入try代码块,但是代码运行中出现死循环或死锁状态
      • 进入try代码块,但是执行了System.exit()操作

JVM(Java虚拟机)

字节码

如何在不同操作系统,不同硬件平台上,均可以不用修改代码执行? <–计算机工程领域的任何问题都可以通过增加一个中间层来解决
Java所有的指令有200个左右,一个字节(8位)可以存256种不同的指令信息,一个这样的字节称为字节码

静态编译将源码转化成字节码以供机器执行

字节码必须通过类加载过程加载到JVM环境后,才可以执行.

执行的三种模式:

  • 解释执行
  • JIT编译执行
    • JIT作用将Java字节码动态地编译成可以直接发生给处理器指令执行的机器码
  • JIT编译与解释混合执型(主流JVM默认执行模式)
    在代码的执行过程中,JVM将字节码解释执行,屏蔽对底层操作系统的依赖;JVM也可以将字节码编译执行,如果是热点代码,会通过JIT动态地编译为机器码,提高执行效率

类加载过程(底层)

ClassLoader加载.class类文件到内存,加载类时使用的是Parent Delegation Model,，主要是在启动之初进行类的Load,Link,Init

  1. Load 阶段读取类文件产生二进制流.并转化为特定的数据结构,初步校验cafe babe 魔法数、常量池、文件长度、是否有父类等，然后创建对应类的 java. Jang.Class 实例。
  2. Link 阶段包括验证、准备、解析三个步骤。验证是更详细的校验，比 final 是否合规、类型是否正确、静态变量是否合理等 准备阶段是为静态变量分配内存,并设定默认值，解析类和方法确保类与类之间的相互引用正确性，完成内存结构布局
  3. Init 阶段执行类构造器＜clinit> 方法，如果赋值运算是通过其他类的静 态方法来完成的，那么会马上解析另外一个类，在虚拟机栈中执行完毕后通过返回值 进行赋值。

一个双向链表与ArrayList 正好相反,适合列表长度位置,添加删除多,尤其两端进行操作,索引位置访问少的情景

两端添加删除效率最高,动态扩展内存平摊,没有索引位置的概念
