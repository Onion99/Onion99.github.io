一个Path对象表示一个文件或者目录的路径，是一个跨操作系统（OS）和文件系统的抽象，目的是在构造路径时不必关注底层操作系统，代码可以在不进行修改的情况下运行在不同的操作系统上。

删除目录树

Files工具类包含大部分我们需要的目录操作和文件操作方法,但Java并没有删除目录树相关的方法

删除目录树的方法实现依赖于 Files.walkFileTree()，”walking” 目录树意味着遍历每个子目录和文件。Visitor设计模式提供了一种标准机制来访问集合中的每个对象，然后你需要提供在每个对象上执行的操作。 此操作的定义取决于实现的 FileVisitor 的四个抽象方法:

  • preVisitDirectory()：在访问目录中条目之前在目录上运行
  • visitFile()：运行目录中的每一个文件
  • visitFileFailed()：调用无法访问的文件
  • postVisitDirectory()：在访问目录中条目之后在目录上运行，包括所有的子目录

文件流

在Java中，文件的输入和输出是通过流（Stream）来实现的。一个流，必有源端和目的端，它们可以是计算机内存的某些区域，也可以是磁盘文件，甚至可以是 Internet 上的某个 URL。

流处理数据单位：

  • 字节流
    • 处理单位是字节，通常用来处理二进制文件，例如音乐、图片文件等
  • 字符流
    • 处理单位是字符，因为Java采用Unicode编码，Java字符流处理的即为Unicode字符，所以在操作汉字、国际化等方面，字符流具有优势
字节流

所有的字节流类都继承自InputStream 和 OutputStream 这两个抽象类

字符流

所有的字符流类都继承自Reader 和 Writer 这两个抽象类，其中Reader是用于读取字符流的抽象类，Writer是用于写入字符流的抽象类

缓冲流

字节流、字符流都是无缓冲的输入、输出流，这容易导致系统负担过重，故不推荐。缓存流是一种装饰器类，目的是让原字节流、字符流新增缓冲的功能。以字符缓冲流为例进行说明，字符缓冲流从字符流中读取、写入字符，不立刻要求系统进行处理，而是缓冲部分字符，从而实现按规定字符数、按行等方式高效的读取或写入
