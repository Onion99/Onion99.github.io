高阶函数

Kotlin语言开始支持面向过程编程，Kotlin语言中可以声明全局函数，内联函数等，还支持函数嵌套，使用函数作为方法参数等操作

高阶函数 , 接受一个或多个过程作为参数,或者以一个过程作为返回结果 ,可以理解为 “以其他函数作为参数或返回结果的函数”

函数类型格式：
(Int) -> Unit

声明规则

  • 通过-> 来表明参数类型和返回值类型
    • 左边是参数类型
    • 右边是返回值类型
  • 必须用（）括号来包裹参数类型
  • 返回值类型即使是Unit，也必须显示声明

示例：

  • 无参函数
    • ( ) -> Unit
  • 多个参数
    • (Int,String) -> Unit
  • 指定参数名字
    • (errCode: Int, errMsg: String) -> Unit
  • 可选参数
    • (errCode: Int, errMsg: String?) -> Unit
  • 可选函数
    • ((errCode: Int, errMsg: String) -> Unit)?
  • 返回另一个函数
    • (Int) -> ((Int) -> Unit)
    • (Int) -> Int -> Unit
  • 先传入一个Int参数,再返回一个Int参数传入,再返回一个Unit
    • ((Int) -> Int) -> Unit

匿名函数

在缺省函数名的情况下，直接定义一个函数,在函数不复用的情况下，相当便利

Lambda

让代码更简单

语法：

  • 通过{ } 包裹表达式
  • 如果声明参数类型,且返回值类型支持推导,Lambda变量就可以省略函数类型声明
  • 如果表达式返回的不是Unit,那么默认返回最后一行代码的值类型
  • 执行表达式方式
    • lambda()()
    • lambda.invoke()

函数与lambda

  • fun在没有等号、只有花括号的情况下，是我们最常见的代码块函数体，如果返回非Unit值，必须带return

  • fun带有等号,是单表达函数体,可以省略return

  • 不管是用val还是fun，如果是等号加花括号的语法，那么构建的就是一个Lambda表达式，Lambda的参数在花括号内部声明。所以，如果左侧是fun，那么就是Lambda表达式函数体，也必须通过（）或invoke来调用Lambda

由这对花括号包裹的代码块如果访问了外部环境变量则被称为一个闭包。一个闭包可以被当作参数传递或者直接使用，它可以简单地看成“访问外部环境变量的函数”。Lambda是Kotlin中最常见的闭包形式。

如果一个函数只有一个参数，且该参数为函数类型，那么在调用该函数时，外面的括号就可以省略

如果参数不止一个，且最后一个参数为函数类型时，就可以采用类似柯里化风格的调用

枚举类和when表达式

枚举是类，相比Java可以拥有构造参数，以及定义额外的属性和方法

当在枚举类中存在额外的方法或属性定义，则必须强制加上分号

用when表达式优化if-else

  • 一个完整的when表达式类似switch语句，由when关键字开始，用花括号包含多个逻辑分支，每个分支由->连接，不再需要switch的break，由上到下匹配，一直匹配完为止，否则执行else分支的逻辑，类似switch的default

  • 每个逻辑分支具有返回值,编译器就会自动推导出每个分支返回值的类型

  • when关键字的参数可以省略

  • 表达式可以组合,整个when表达式只会匹配成功一个

构造对象

Kotlin采用新的构造方法来解决Java中的构造方法过多,代码冗余的问题

声明:

  • 构造方法默认参数

  • init语句块,用在初始化时进行其他的额外操作

    • 当没有val或var的时候，构造方法的参数可以在init语句块被直接调用,除此之外不能

  • 延迟初始化: by lazy 和 lateinit

    • by lazy 必须用于val声明的变量,首次调用才赋值
    • lateinit主要用于var声明的变量，然而它不能用于基本数据类型，需要用Integer这种包装类作为替代

密封类

除了可以利用final来限制一个类的继承以外，还可以通过密封类的语法来限制一个类的继承

Kotlin通过sealed关键字来修饰一个类为密封类，若要继承则需要将子类定义在同一个文件中，其他文件中的类将无法继承它,但这种方式有它的局限性，即它不能被初始化，因为它背后是基于一个抽象类实现的

访问控制

要么为继承做好设计并且提供文档，否则就禁止这样做

  • ·Kotlin中没有采用Java中的extends和implements关键词，而是使用“：”来代替类的继承和接口实现
  • 由于Kotlin中类和方法默认是不可被继承或重写的，所以必须加上open修饰符

Kotlin 与 Java的限制修饰符比较

修饰符 含义 与Java比较
open 允许被重写或继承 相当于java类与方法的默认情况
abstract 抽象类或抽象方法 与Java一致
final 不允许被继承或重写(默认情况) 与Java一致

Kotlin 与 Java的可见性修饰符比较

修饰符 含义 与Java比较
public 默认修饰符,全局可见 与Java一致
protected 受保护修饰符,类及子类可见 含义一致,但作用域除类及子类外,包内也可见
private 私有修饰符,类内修饰,只有本类可见,类外修饰,文件内可见 私有修饰,只有类内可见
internal 模块内可见 无

数据类

单纯地使用类来封装数据,相对Java来说,不须重写equals,hashcode,toString,get/set,直接进行判等,自带copy与componentN方法

  • 数据类必须拥有一个构造方法，该方法至少包含一个参数，一个没有数据的数据类是没有任何用处的
  • 与普通的类不同，数据类构造方法的参数强制使用var或者val进行声明
  • ·data class之前不能用abstract、open、sealed或者inner进行修饰

伴生对象

“伴生”是相较于一个类而言的，意为伴随某个类的对象，它属于这个类所有，因此伴生对象跟Java中static修饰效果性质一样，全局只有一个单例。它需要声明在类的内部，在类被装载时会被初始化

companion object用花括号包裹了所有静态属性和方法，使得它可以与Prize类的普通方法和属性清晰地区分开来。
